## 🎤 Сценарий защиты: формат подгруппы 3–4 человека

### Регламент

| Этап | Время | Кто | Что делает |
|------|-------|-----|------------|
| **Выступление 1** | 5 мин | Участник 1 | Слайды 1–3: тема, нормативная база, технологии тьюторского сопровождения |
| **Выступление 2** | 5 мин | Участник 2 | Слайды 4–5: структура ЭУП, контроль и оценка |
| **Выступление 3** | 5 мин | Участник 3 | Слайды 6–7: взаимодействие с работодателями, ресурсы и время |
| **Обсуждение в подгруппе** | 5 мин | Все участники | Заполнение слайда 8, обмен мнениями |
| **Отчёт руководителя** | 5 мин | Руководитель | Слайд 8: оценка каждого участника, рекомендации |
| **Выступление для гостей** | 3 мин | Любой участник | Краткий обзор программы |

> **Итого на подгруппу:** ~15–20 мин выступлений + 5 мин подготовка отчёта.

### Распределение слайдов между участниками

| Участник | Слайды | Тема | Время |
|----------|--------|------|-------|
| **Участник 1** | 1–3 | Введение + нормативная база + технологии | ~5 мин |
| **Участник 2** | 4–5 | Структура ЭУП + контроль и оценка | ~5 мин |
| **Участник 3** | 6–7 | Работодатели + ресурсы + перспективы | ~5 мин |
| **Руководитель** | 8 | Отчёт подгруппы | ~5 мин |

#### Слайд 5 — Контроль и оценка (2,5 мин)

> **Текст:**
>
> **Система контроля и оценки** реализована на нескольких уровнях.
>
> **Первый уровень — тестовые задания.** В разделе «Тесты» представлено **10 вопросов трёх типов**:
> - *Одиночный выбор* — 6 вопросов
> - *Множественный выбор* — 3 вопроса
> - *Сопоставление* — 1 вопрос
>
> Проверка выполняется **автоматически** при нажатии кнопки «Проверить все ответы». Система подсчитывает количество правильных ответов, вычисляет процент и выставляет оценку по шкале:
> - 90%+ — «Отлично»
> - 70–89% — «Хорошо»
> - 50–69% — «Удовлетворительно»
> - Менее 50% — «Требуется повторение»
>
> По каждому вопросу выводится **обратная связь**: ✅ «Верно» или ❌ «Неверно» с подсказкой.
>
> **Второй уровень — статистика в localStorage.** Все результаты тестов сохраняются в браузере: дата, количество баллов, процент. Пользователь может отслеживать **динамику своих результатов** и средний балл. Также отмечаются посещённые страницы.
>
> **Третий уровень — чек-листы и критерии оценки.** В разделе «Рекомендации тьютора» представлен чек-лист готовности к защите проекта, а также модель оценки эффективности тьюторского сопровождения **по Киркпатрику** (4 уровня: реакция, усвоение, поведение, результат).
>
> **Пример:** если студент прошёл тест 3 раза и его результат вырос с 50% до 80%, это видно в статистике. Такая динамика — показатель эффективности усвоения материала.
>

## 🚀 Как запустить проект

### Локальный запуск
Просто откройте `index.html` в любом современном браузере. Все пути относительные, сервер не требуется.

### Публикация на GitHub Pages
1. Создайте репозиторий на GitHub
2. Загрузите все файлы проекта
3. В настройках репозитория включите GitHub Pages (ветка `main`, корневая папка `/`)
4. Через 1–2 минуты сайт будет доступен по адресу: `https://[username].github.io/[repository]/`
